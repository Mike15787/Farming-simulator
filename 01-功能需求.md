# 種田遊戲 — 功能需求規格（MVP）

## 專案概述

一款基於 **Electron + Phaser.js** 的 2D 種田遊戲。初版（MVP）以驗證核心玩法循環與劇情任務節奏為目標，美術全面採用**色塊 / 簡單幾何圖形**表示物件，不依賴任何外部 sprite 素材，確保低算力筆電也能流暢運行。

**開發週期**：1–2 週
**技術取向**：邏輯吃重、美術從簡、零 AI 運算
**核心循環**：種田 → 時間推進 → 收成 → 完成 NPC 任務 → 進出房屋場景

---

## 1. 地圖與田地系統

### 1.1 戶外農場地圖
- 固定大小格子地圖，建議 **15×15** 格（每格 32×32 px）。
- 格子地形類型：
  - **草地**（綠色 `#7CB342`）：不可耕種，可行走。
  - **可耕地**（淺褐 `#A1887F`）：可翻土種植。
  - **水域**（藍色 `#4FC3F7`）：不可行走，作為視覺與邊界。
  - **房屋區**（含門）：碰觸門觸發場景切換。
- 地圖以二維陣列定義，每格記錄地形類型與其上的物件狀態。

### 1.2 田地狀態機
單一耕地格的狀態流轉：

```
空地(可耕地) → 翻土 → 播種 → 已澆水 → 成熟 → 收成 → 回到翻土狀態
```

- **翻土**：將可耕地轉為可播種的耕地（褐色 `#8D6E63`）。
- **播種**：在翻過的土上放下種子，作物進入生長階段 0。
- **澆水**：當天澆水後，作物在隔天才會推進生長階段（深褐 `#6D4C41` 表示濕潤）。
- **成熟**：達到最終生長階段，可收成。
- **收成**：取得作物進入背包，格子回到翻土（未播種）狀態。

## 2. 作物生長系統

- 目前共 **6 種作物**:番茄(初版起始)、胡蘿蔔、馬鈴薯、玉米、南瓜、高麗菜。新蔬菜種子於商店販售(見第 9 節)。
- 所有作物共用同一套狀態機與生長規則(差別只在名稱、價格與成熟果實顏色)。
- 生長階段共 **4 段**：種子 → 發芽 → 成長 → 成熟。
- 推進條件：**當天有澆水** + **進入下一天**，缺一不可。
- 視覺表示（色塊 / 幾何形狀變化）：
  - 種子：土上一個小點。
  - 發芽：一條短綠線。
  - 成長：較高的綠線 + 分岔。
  - 成熟：綠線頂端加一個彩色圓(果實),**顏色依作物種類不同**。
- **枯萎機制**：MVP 階段先**不做**枯萎，降低複雜度（列為未來擴充）。

## 3. 時間系統

- 遊戲內以「天」為基本單位。
- 不做即時時鐘，改用**事件驅動推進**：玩家回到室內、碰床「睡覺」即結束當天、進入下一天。
- 每次進入新一天時，統一結算所有作物的生長（檢查前一天是否澆水）。
- 畫面上以簡單 UI 顯示「第 N 天」。
- （未來擴充：日夜循環、即時時段、季節。）

## 4. 玩家移動與互動

> 註：此節已於初版設計後修訂。原設計為「逐格移動 + 面前一格互動」，實際採用**自由移動 + 最近目標互動**(下述)，操作更直覺、不需對齊格子。

- **移動**：方向鍵 / WASD，**自由像素移動(XY 軸)**，不以格子為單位、可斜向移動。撞到不可行走的格子(水域 / 牆)會被擋下,可沿牆滑行。
- **碰撞**：以玩家方框是否壓到「實心格」判定;水域、牆、NPC、地圖界外為實心。
- **互動方式**：單一**動作鍵**(空白鍵 / E)或畫面上的**動作按鈕**(滑鼠點擊)。動作作用於玩家附近(可及距離 `REACH` 內)**最近一個「有事可做」的田地格**,系統自動依該格狀態決定動作:
  - 可耕地 → 鋤地。
  - 翻過的土(且有選取種子) → 播種。
  - 已播種未澆水 → 澆水。
  - 成熟作物 → 收割。
- **目標提示**：畫面會**高亮**目前鎖定的目標格,動作按鈕也即時顯示「現在會做什麼」(鋤地 / 播種 / 澆水 / 收割);玩家不需對齊格子也能預期作用對象。
- **NPC / 門**:靠近 NPC(`REACH` 內)時,動作鍵 / 按鈕改為**開啟對話**;走到「門」格即**碰觸觸發**場景切換。
- 玩家面向(facing)仍會記錄,但僅作為視覺朝向(角色的小方塊指示),不再用於互動判定。

## 5. 背包與物品系統

- 簡化的格子式背包（例：8–12 格）。
- 物品類型：種子、收成的作物、（未來）工具。
- 每個物品有 id、名稱、數量、圖示色塊。
- MVP 的 UI：畫面底部一排格子，顯示物品色塊 + 數量。
- 需有「當前選取物品」概念（決定播種時種下哪種種子）。

## 6. 場景切換系統（房屋進出）

- 至少兩個場景：**戶外農場** 與 **室內房屋**。
- 戶外房屋有一個「門」格子，玩家碰觸 → 切換到室內場景。
- 室內有出口門，碰觸 → 切回戶外，並還原玩家在戶外門口的位置。
- 室內初版陳設（色塊）：
  - **床**：碰觸 → 睡覺 → 進入下一天。
  - **地板 / 牆壁**：色塊區分可行走範圍。
  - （未來：櫃子 = 倉庫、其他家具。）
- 切換時需正確傳遞「玩家應出現的座標」。

## 7. NPC 與劇情任務系統

- MVP 階段先做 **1 個 NPC**。
- 對話系統：底部對話框，逐句顯示文字，按鍵推進。
- 任務範例：「幫我帶 3 個番茄來」。
  - 接任務 → 種植收成 → 背包達標後再對話 → 交付 → 給獎勵（金錢 / 種子）。
- 任務狀態：未接取 → 進行中 → 可交付 → 已完成。
- NPC 對話內容依任務狀態變化。

## 8. 存檔系統

- 使用 **localStorage** 序列化遊戲狀態為 JSON。
- 存檔內容：玩家座標(像素)與當前場景、天數、田地狀態陣列、背包內容、金錢、任務狀態,以及存檔版本號。
- 時機：每天結束（睡覺）時自動存檔;切換場景(進出房屋)時也存檔。
- 啟動時若有存檔則載入，否則開新局;舊版存檔(玩家座標為格子)會在載入時自動遷移為像素座標。

## 9. 商店與市場系統

戶外地圖上有兩個色塊建物,靠近後用動作鍵 / 動作按鈕開啟對應的模態面板:

### 9.1 商店(購買種子)
- 商店色塊(藍紫,標「商」)。販售 **5 種蔬菜種子**:胡蘿蔔、馬鈴薯、玉米、南瓜、高麗菜(價格暫定,可調)。
- 面板列出每種種子的色塊、名稱、單價與「購買」鈕;點擊購買一份 → 扣金錢、種子入背包。金錢不足時拒絕並提示。

### 9.2 市場(販售作物)
- 市場色塊(青綠,標「市」)。收購背包內的**作物(蔬菜水果)**換取金錢。
- 面板列出背包中可賣的作物、持有數量與**動態市價 + 漲跌箭頭(對比基準價)**,提供「賣 1」與「全部賣」。
- **價格隨供需浮動**(見第 10 節):每賣 1 個就把該作物供給 +1,價格即時下修 → 一次倒貨會邊際遞減。
- 種子不可販售;沒有可賣作物時顯示提示。

### 9.3 共通
- 面板蓋住地圖區,但保留底部 UI 列(可同時看到金錢與背包)。按 Esc / ✕ / 點 UI 列即關閉。
- 開啟期間暫停世界(玩家不動、世界互動停用),所有點擊交給面板。
- 每次交易與關閉時自動存檔。
- **經濟循環**:賣作物賺錢 → 到商店買新種子 → 種植收成 → 再販售,形成正向循環。

## 10. 多農夫競爭與動態市場

把遊戲做成「傳說中的農夫」式的競爭:你不是市場上唯一的供給者,要在價格波動中比其他農夫賺更多。

### 10.1 NPC 競爭農夫(模擬農場)
- 數個 NPC 農夫(初版 3 位),各自以**彩色方塊 + 名字**站在地圖上,可靠近攀談看狀態(在種什麼、現價、身家、排名)。
- 每位農夫有**固定的種植流程**(routine,作物循環)與成長天數、單次產量;**田不畫在地圖上**,以背景模擬。
- 每天結算時,到期的農夫會**收成 → 把產量灌入市場供給**(壓低該作物價格),並把收成換成自己的財富。

### 10.2 動態定價(供需 + 漲跌上限 + 每日胃納)
- 每種作物有一個**市場需求量上限 demand** 與會累積的**供給庫存 supply**。
- 成交價公式:`Price = BasePrice × (1 + SupplyFactor) × Appetite`,其中
  `SupplyFactor = clamp( -K × (supply/demand − 1), −MAX_DROP, +MAX_RISE )`。
  - 供 = 需 → 係數 0;供 > 需 → 下跌;供 < 需(稀缺)→ 上漲。**漲跌有上限**:最低 `BasePrice×(1−MAX_DROP)`(地板),最高 `BasePrice×(1+MAX_RISE)`(天花板),再乘 Appetite。
  - 預設 `K=0.5`、`MAX_DROP=0.6`(地板=基準價 40%)、`MAX_RISE=0.4`(天花板=基準價 140%),皆可在 config 調整。
- **Appetite(今日市場胃納)** = 每日波動 × 節慶加成,價格與需求消耗都乘上它(見 10.3)。
- **供給來源**:NPC 農夫每日收成 + 玩家賣出(每賣 1 個 supply +1)。
- **需求消耗(具價格彈性)**:每天結算時,每作物 `supply -= 當日需求`,其中
  `當日需求 = round( demand × Appetite × (1 − ELASTICITY × SupplyFactor) )`(不為負)。
  - 價低(供過於求,SupplyFactor<0)→ 當日需求大於 demand,加速消化庫存、價格更快回穩;價高(稀缺)→ 消耗較少。
  - 預設 `ELASTICITY=0.8`(設 0 即退化為固定吃 demand),可在 config 調整。供給設上限 `demand×4` 避免回升過久。

### 10.3 每日需求波動 + 節慶
- **每日波動**:每天每作物的 Appetite 有一個隨機倍數(預設 ±`DEMAND_NOISE=20%`),讓行情天天有高低。倍數以「(天數+作物)為種子」的**確定性偽隨機**產生 → 同一天固定、不閃爍、**不需存檔**。
- **節慶**:每 `FESTIVAL_CYCLE`(預設 30)天循環一次行事曆。節慶當天,對應作物 Appetite ×`mult`(預設 3),**需求量與成交價同步暴增**,適合平日囤貨、在節慶當天大量賣出(需求大 → 一次倒貨也不易崩盤)。
  - 內建:**🍅 番茄節**(循環第 15 天 → 番茄)、**🎃 萬聖節**(第 25 天 → 南瓜)。新增/調整節慶只改 `config.FESTIVALS`。全部由天數決定,**不需 migrate**。
  - **提示**:頂列置中顯示節慶徽章(農場中也看得到);市場面板顯示節慶橫幅,熱門作物名稱前加 emoji。

### 10.4 財富排行榜
- 排行榜公佈欄色塊(暗金,標「榜」),靠近開啟唯讀面板。
- 列出**玩家與所有 NPC 農夫**的財富,由高到低排序,標出玩家、冠軍加 👑。
- 目標:成為財富第一的「傳說中的農夫」。

## 11. 日期與天氣系統

### 11.1 日期系統
- 遊戲從 **827 年 11 月 9 日** 開始;睡覺過一天,日期 +1(年/月進位;2 月固定 28 天,忽略閏年)。
- 日期由 `day` 計數器推導(不另存存檔);頂列 HUD 顯示「827年11月9日」。
- 季節(每 3 個月一季):冬(11–1)、春(2–4)、夏(5–7)、秋(8–10)。季節決定天氣機率。

### 11.2 天氣系統
- 天氣種類:**晴天 ☀️ / 小雨 🌦️ / 大雨 🌧️ / 下雪 ❄️ / 颱風 🌀**。每天分**上午、下午**兩段。
- 依季節出現:冬(晴/小雨/雪)、春(晴/小雨/大雨)、夏(晴/小雨/大雨/**颱風**)、秋(晴/小雨/大雨)。
- 天氣為「每局固定」:由一個 `weather.seed` + 日期確定性推導(每局不同、可預測未來、跨讀檔穩定)。
- **對作物的影響**:
  - 小雨/大雨/颱風當天**自動幫所有作物澆水**(不用自己澆);晴天/多雲/雪要自己澆。
  - **下雪不影響生長**(只是天氣);**颱風**造成農損:未受保護的作物**掉 1 生長階、當天不生長**。
  - 颱風還會**衝擊市場**(全作物供給下修 → 價格上揚)、隔天起有幾天「颱風尾」的雨。

### 11.3 天氣預報(布告欄)
- 天氣預報布告欄色塊(青,標「報」),靠近開啟唯讀面板,列**未來 14 天**(今天 + 13 天)。
- **越遠越不準**:當天準確率 100%,每往後一天 −5%(第 14 天約 30%);不準時顯示的是同季節的合理假天氣。隨天數接近會逐步「揭露」成真天氣。
- **颱風預警**:真颱風在來襲**前 7 天**才於預報顯示「颱風警報」,更早只看得到「大雨」。HUD 也會在 7 天內顯示颱風警報徽章。

### 11.4 防災對策
- **棚子**(逐格):在商店買「棚子」,手持它對土格搭建;保護該格作物不受颱風農損。**不擋雨**(雨照樣自動澆水),但強颱有機率把棚子吹壞需重搭。
- **魔法結界**(全域):在天氣預報面板花錢施放,保護**整片農場** N 天(期間免疫颱風),但不擋雨。適合預報看到颱風時應急。
- 玩家看預報 → 判斷颱風 → 用棚子(平時逐格布置)或結界(臨時全域)做對策。
